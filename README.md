# ilyalebe.dev

Source for [ilyalebe.dev](https://ilyalebe.dev), the personal website of Ilya Lebedev. Bio, four CV variants for different audiences, project highlights, a travel map, and a [contact page](https://ilyalebe.dev/contact) wired to Calendly.

## Stack

- [Astro 5](https://astro.build) (static site generation, View Transitions)
- [Tailwind CSS](https://tailwindcss.com) + [DaisyUI](https://daisyui.com) v5
- [MapLibre GL](https://maplibre.org) + [d3-geo](https://github.com/d3/d3-geo) for the travel page
- TypeScript

## What's interesting in here

- **Multi-flavour CV.** Same career re-cut for different audiences: [general](https://ilyalebe.dev/cv), [fullstack](https://ilyalebe.dev/cv_fullstack), [backend](https://ilyalebe.dev/cv_backend), and [PM](https://ilyalebe.dev/cv_pm).
- **AI-driven discovery.** [`public/llms.txt`](./public/llms.txt) follows the [llmstxt.org](https://llmstxt.org) spec so answer engines (Perplexity, ChatGPT search, Claude, etc.) can index the canonical bio and CV variants without scraping HTML.
- **Structured data.** JSON-LD `Person` + `WebSite` + `ProfilePage` (`@graph`) emitted from `BaseHead.astro` so search engines and rich profile cards have a well-typed entity to point at.
- **CI link check.** Every push runs Lychee against the built `dist/` and fails on broken external links (with LinkedIn's 999 anti-bot status filtered out).
- **View Transitions.** Page navigation cross-fades via Astro's `<ClientRouter />`. The travel map re-initialises cleanly on `astro:page-load`; the sidebar active state is computed server-side per page render.
- **Offline travel data.** Countries and cities live in [`src/data/travel.ts`](./src/data/travel.ts); the world map renders client-side from `world-atlas` TopoJSON via MapLibre.

## Local development

```sh
npm install
npm run dev       # http://localhost:4321
npm run build     # static output to dist/
npm run preview   # serve the built dist/
npx astro check   # typecheck + Astro lint
```

## Project layout

```
.github/workflows/  CI: astro check + build + lychee link check
public/             Static assets: images, llms.txt, robots.txt, profile.png
src/
  components/         Sidebar, header, footer, cards, travel map, BaseHead
  layouts/            BaseLayout
  pages/              index, cv, cv_{fullstack,backend,pm}, contact, travel, 404
  data/               Static data (visited countries)
  styles/             Global CSS
  config.ts           SITE_TITLE / SITE_DESCRIPTION
```

The sitemap is auto-generated by `@astrojs/sitemap` at build time (`/sitemap-index.xml` + `/sitemap-0.xml`); `public/robots.txt` points at it.
